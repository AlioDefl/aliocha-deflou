Trading Bot Crypto.

Webapp de paper trading crypto algorithmique : choix d'une paire, paramétrage de stratégies (SMA, RSI, Bollinger), backtest sur historique et simulation live via WebSocket. Projet de fin de BUT2 documenté en ADR.

Le contexte.

Projet de fin d'année BUT2 Informatique (parcours Réalisation d'Applications, IUT de Lille). L'objectif : démontrer une maîtrise fullstack complète sur un domaine métier non-trivial — le trading algorithmique — sans jamais manipuler d'argent réel (paper trading uniquement).

Le projet doit montrer la capacité à : intégrer une API externe (ccxt pour les données de marché), concevoir une persistance fiable (PostgreSQL avec migrations), implémenter de la communication temps réel (WebSocket), et documenter ses choix d'architecture (ADRs).

La stack.

FrontendReact 18 + TypeScript
StylingTailwind CSS + Recharts
BackendPython 3.11 + FastAPI
Tradingccxt + pandas + numpy
BDDPostgreSQL + SQLAlchemy + Alembic
InfraDocker Compose

Stack figée par ADR-001 avant le moindre commit de code. Architecture détaillée dans ADR-002.

L'architecture.

Backend FastAPI exposant une API REST + un endpoint WebSocket. Le frontend React consomme l'API pour les écrans de configuration (paire, stratégie, paramètres) et s'abonne au WebSocket pour la simulation live. Persistance PostgreSQL pour les configurations et l'historique des backtests.

Les stratégies (SMA, RSI, Bollinger) sont implémentées en pandas/numpy côté backend, ce qui permet de réutiliser le même code pour le backtest historique et la simulation live.

Les défis.

  • Concevoir un moteur de backtest qui produit les mêmes résultats que la simulation live (cohérence). Solution : code de calcul d'indicateurs unifié, alimenté par un flux de bougies (historique ou temps réel).
  • Gérer les rate limits de l'API ccxt sans bloquer l'UI. Solution : worker async côté backend, le frontend ne fait que lire la dernière donnée disponible.
  • Documenter les choix techniques pour qu'ils restent compréhensibles dans 6 mois. Solution : ADRs systématiques pour chaque décision structurelle.

Les apprentissages.

Architecture asynchrone Python (async/await), ORM avec migrations versionnées (Alembic), conteneurisation multi-services (Docker Compose), documentation par décisions architecturales (ADR), et la valeur de figer la stack avant de commencer à coder.
